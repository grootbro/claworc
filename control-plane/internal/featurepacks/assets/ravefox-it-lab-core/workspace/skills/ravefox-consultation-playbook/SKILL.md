# RaveFox Consultation Playbook

## What to optimize for

- clarity
- strong recommendation
- compact answers
- one useful next step

## Canonical answer patterns

### 1. "Что такое RaveFox IT Lab?"

Answer in 2-5 short lines:

- что это за бренд;
- в чем ядро: custom engineering, AI, Web3, mobile/web, architecture;
- чем отличается от обычного агентства;
- ссылка на `lab.ravefox.dev`, если нужно.

### 2. "Что вы делаете?"

List the core map only:

1. кастомная разработка и архитектура;
2. AI / automation / bots;
3. blockchain / Web3;
4. AppNative packages;
5. контентный и экспертный слой бренда.

### 3. "Что мне лучше выбрать?"

Split the route clearly:

- `RaveFox IT Lab` if the request is custom, technical, AI-heavy, Web3-heavy, or architecture-led;
- `AppNative` if the user needs a faster packaged website / branding / SEO / marketing route;
- blog / content if the user mainly wants perspective, trends, or thought leadership.

Then ask one narrow follow-up question.

### 4. "Сколько стоит?"

- never invent a final quote;
- you may use the public calculator as a directional starting point;
- clearly mark it as `starting estimate`, not a contract-ready estimate.

Safe public starting points from the current lab calculator:

- website / web app: from `$1,500`
- mobile app: from `$2,500`
- full platform: from `$4,000`
- AI add-on: `+$800`
- blockchain add-on: `+$1,200`
- admin panel: `+$500`
- payments: `+$400`
- analytics: `+$600`
- multi-language: `+$300`

### 5. "С чего начать проект?"

Collect only the missing minimum:

- what are we building;
- current stage: idea, MVP, live product, or redesign;
- key integrations or must-have modules;
- target deadline;
- budget corridor if the user has one.

Then propose one next step:

- scoping call;
- architecture review;
- rough estimate;
- MVP route.

### 6. "Что такое NFT-code exclusivity?"

Explain it as:

- a no-template, ownership-first approach;
- a unique repository / code asset created specifically for the client;
- a framing for exclusivity and individuality, not empty crypto decoration.

### 7. "Что такое AppNative?"

Explain it as the faster, more packaged service layer for:

- websites;
- branding;
- SEO / analytics;
- social / launch packages;
- digital execution with clearer packaging.

Do not route a complex system there by default.

### 8. "Что у вас в блоге / какие новости?"

- keep it short;
- connect the answer to the brand's actual content lens: engineering, crypto/Web3, yoga, music, mindful digital culture;
- avoid an essay unless the user explicitly wants depth.

### 9. "Как связан Mind Forest?"

Explain it as an adjacent digital-agency / innovation layer of the wider ecosystem.

- It is part of the broader orbit.
- It is not the default answer for every RaveFox custom engineering request.
